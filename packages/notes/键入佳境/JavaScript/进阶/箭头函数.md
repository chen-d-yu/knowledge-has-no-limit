# 箭头函数(arrow function)

声明函数的方式有三种
- 函数表达式
- 函数式声明
- 箭头函数

箭头函数的写法比另外两种更加简洁

```js
// 1.函数表达式
var fn = function(){}

// 2.函数式声明
function foo(){}

// 3.箭头函数，()参数列表，{}方法体，用箭头链接
var arrowFun = () => {}
```

## 基本语法

关于箭头函数，有一些省略的写法，关于参数和返回值的
1. 如果箭头函数只有一个参数，可省略参数括号
2. 如果代码体中只有一句代码，可以省略大括号
3. 如果该箭头函数返回的是对象，需要用括号包裹

```js
// 1.省略参数括号
var fn1 = a => {};

// 2.省略大括号
var sum = (x, y) => x + y

//3.省略大括号，并且返回的是对象
var foo = (name) => ({name:name})
```

## 箭头函数的特点

### 1. 不创建自己的this

箭头函数不会创建自己的`this`，它会沿着作用域链向上寻找`this`

没有`this`，自然无法改变`this`的指向，所以`apply/call/bind`等方法不起效

也就说明，箭头函数的`this`确认，是在函数定义环节就已经确定了，它的`this`指向永远不会发生改变

```js

var obj = {
	name: "张三",
	foo: () => {
		console.log(this)
	}
}

// 1.箭头函数内不创建this，只能向上寻找，而obj字面量{}不会形成作用域，所以执行找到并输出window
obj.foo()

var person = {
	name: "孙悟空"
}

// 2.箭头函数没有this，所以这些方法对箭头函数无效，this还是按照作用域链查找的方式，执行并输出window
obj.foo.apply(person)

```

> 对象字面量不会形成作用域
> 对象字面的大括号仅仅表示创建对象的语法糖，它不会形成作用域

### 2. 箭头函数没有arguments对象

普通函数中存在一个隐式的`arguments`类数组对象，但是在箭头函数中却不会绑定这个对象

`ES6`之后，有了剩余参数的语法，来替代箭头函数内的`arguments`对象

```js
var testArguments = () => {
	console.log(arguments)
}

// 1.不会绑定arguments，一旦调用，在自身找不到该对象，就会沿着作用域链向上寻找，找到全局都没有，就会报错 arguments is not defined
testArguments()

// 2.使用剩余参数替代arguments，args是个真正的数组，接收testArgs函数的剩余参数，执行输出 args => [1,2,3,4]
var testArgs = (...args) => {
	console.log(args)
}

testArgs(1,2,3,4)
```


### 3. 箭头函数无法作为构造函数

箭头函数是一个特殊的函数对象，众所周知是个对象都有它的隐式原型`__proto__`，函数对象不仅有隐式的，还多一个显示原型`prototype`——[[大话原型链]]
箭头函数特殊的点就在于，它是没有显示原型的，箭头函数没有`prototype`属性

![[Pasted image 20230506142335.png]]

由于它没有`prototype`属性，在作为构造函数时，执行[[new关键字]]操作它时，就会出现错误从而抛出`Uncaught TypeError: fn is not a constructor`

## 总结

1. 箭头函数可以在参数、方法体有一些省略写法
2. 箭头函数没有`this`，它的`this`指向在定义阶段已经确定了，是继承于上层作用域的`this`，所以无法被`apply/call/bind`等方法修改`this`指向，会无效
3. 箭头函数没有`arguments`，在`ES6`中用剩余参数代替`arguments`
4. 箭头函数是个特殊的函数对象，它没有显示原型`prototype`，所以它无法当做构造函数，无法被`new`关键字所操作
