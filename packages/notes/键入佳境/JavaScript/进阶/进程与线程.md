# 进程与线程

>进程和线程是操作系统里面的概念，通过系统内核的调度，完成多任务并行的操作

## 进程`Process`

进程是计算机分配的最小资源单位，进程为线程提供资源，可以说进程就是一个容器，提供程序运行时的内存资源

计算机会为每个进程分配不同的内存空间，程序的每一次执行，都会分配一块独立的内存空间，用实际例子来说，比如打开QQ，那么系统就会为QQ分配资源空间，就相当于开启一个进程

## 线程`Thread`

是计算机中可以分配`cpu`调度的最小工作单位，也是程序实际工作单位

线程是依附进程去工作的，一个进程中可以有多个线程，同一个进程中的线程，共享当前进程中的资源

在进程中可以使用多线程并行处理，可以提升运算效率

## 进程和线程的联系和区别

1. 进程和进程之间是完全隔离的，互不干扰，一个进程崩溃不影响整个程序
2. 进程与进程之间通信是依赖`进程通信管道IPC`来完成的
3. 一个进程内可以并发多个线程，各个线程执行不同的任务，这就是多线程
4. 同个进程下的各个线程，可以直接进行通信和共享进程资源
5. 当进程关闭时，操作系统会回收分配的资源

**简单点来说：**

一个公司架构可以看做一个进程，公司的各个部门看做是每个线程，公司给各个部门提供资金，各个部门去完成今年的任务——进程提供资源给线程，线程执行不同的任务，这就是进程与线程的现实例子

何谓单线程、多线程？

单线程就是这个公司只有你一个人，你需要负责整个公司所有事务，当想和其他的公司进行交流，只能先与别的公司预定好（`进程通信管道IPC`）才能进行交流

多线程是，公司有不同的部门，各个部门可以直接跨部门通信，共享公司的客户资源，这些部门负责不同的任务，提高公司运作效率
