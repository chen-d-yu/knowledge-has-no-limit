# 组件化开发

## 没有组件化时，怎么开发的？

比如现在的需求正在编写一个网页，这个网页分为上中下：导航栏、中间内容区域、页脚，

开发人员 A 开发完之后，现在又来了个需求，写一多一个网页，结构分为上中(左右)下布局：导航栏、中间：sidebar-content、页脚 footer，那就需要重新编写结构、样式、交互

如果下次再来这样的需求，就需要重复编写

一个页面作为一个单位开发，开发单位过大，复杂度上升

一个页面可能出现大多数功能逻辑相似的地方，很难提取公共的区域

这个时候就需要组件化开发，那什么是组件化开发呢？

## 组件化

先简单理解为，组件就是界面中抽离的区域，比如上面的网页，分为三个区域：上 header、内容 content、页脚 footer

将这三个区域抽离成组件，这三个组件下或许又抽离成了组件，这样这些组件之间就形成了一颗组件树

那组件到底是什么呢？**就是实现应用中区域功能代码和资源的集合抽离、封装，只要引入就能使用，哪里需要哪里搬**

组件化开发的意义在于：

- 降低开发的粒度，整体降低复杂度
- 减少重复代码，提升了开发效率和可维护性
- 抽离成为公共的库或者公共组件

## 非单文件组件

一个文件中包含多个组件，利用 Vue.extends api 去创建一个组件实例，并挂载到 root 组件下

在组件文件中，data 一定要配置为函数，不然报错

## 单文件组件

一个文件只能包含一个组件
