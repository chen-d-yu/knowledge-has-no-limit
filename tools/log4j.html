Log4js ​

日志的概念 ​

日志文件是用于记录系统操作事件的文件集合，可分为事件日志和消息日志。具有处理历史数据、诊断问题的追踪以及理解系统的活动等重要作用。

日志的作用主要是以下几个方面的：

  • 调试：可以方便知道当前程序的运行状态，输出的日志便于记录程序在之前的运行结果。
  • 错误定位：项目在运行一段时候后，可能由于数据问题，网络问题，内存问题等出现异常。这时日志可以帮助开发或者运维人员快速定位错误位置，提出解决方案。
  • 数据分析：收集用户行为，对这些数据进行分析，形成用户画像。

主流日志技术

在 Node 服务端的日志收集工具中，比较火热的有 winston、log4js、bunyan、npmlog 等，对于日志收集的基本要求例如分布式集中收集、多渠道输出、日志格式化等上述 npm 库均可满足。

至于为什么学习 log4js ，那当然是因为它的顶顶大名。

基本使用 ​

安装 ​

shell
pnpm install log4js -S
1

使用 ​

最简单的使用方式：

ts
import log4js from "log4js";

// 获取log4j的实例
const logger = log4js.getLogger();

// 默认的级别是 OFF，表示不打印任何日志，这里修改级别为debug，表示打印debug以下的日志
logger.level = "debug";

logger.debug("Some debug messages");
logger.info("Some info messages");
logger.warn("Some warning messages");

configure——配置 ​

log4j 提供了配置接口，来配置我们想要的输出源—— appenders 。log4j.configure() 可以接受文件路径，也可以接受配置对象。

如果你不进行配置，那么 log4j 会启用默认配置。默认配置定义了一个 appenders ，它将会使用彩色的样式进行标准输出，但是也会将日志级别设置为 OFF 。也就是说，即时不进行配置，你也需要调整日志级别。

配置有三个核心概念：

  • levels：日志级别，默认情况下，ALL < TRACE < DEBUG < INFO < WARN < ERROR < FATAL < MARK < OFF，只会打印大于等于 logger 实例设置的日志级别的日志。
  • appenders：日志输出源，日志输出到哪里，由什么样的方式输出，由 appenders 来决定。
  • categories：将日志输出源进行分组，是为了让日志在级别区分之外，还有分组区分的维度。
js
/**
 * @type {import("log4js").Configuration}
 */
const config = {
  // 输出源
  appenders: {
    // 定义一个out的输出源，type表示将日志输出到控制台
    out: {
      type: "console"
    },
    app: {
      type: "console"
    }
  },
  // 定义实例日志级别，如果要在config中配置的话，需要添加上自定义样式以及级别数值，数值越大，级别越高
  // 默认情况下，ALL < TRACE < DEBUG < INFO < WARN < ERROR < FATAL < MARK < OFF
  // OFF 表示关闭日志打印，而不是作为实际的日志级别，所以你不能调用 logger.OFF('some message') 打印日志
  levels: {},
  // 将日志输出源进行分组，可以使用 log4js.getLogger('app') 来获取对应的分组
  // 分组的作用，为了让输出日志进行分组区分日志，
  // 比如默认的输出日志：[2024-02-05T15:28:12.820] [DEBUG] default - Some debug messages
  // 获取到debug分组的日志：[2024-02-05T15:28:12.820] [DEBUG] debug - Some debug messages
  // 他们的区别，就在于[DEBUG]的打印级别后面的 default 和 debug的区分
  categories: {
    default: {
      appenders: ["out"],
      level: "DEBUG"
    },
    app: {
      appenders: ["app"],
      level: "DEBUG"
    }
  }
};

export default config;

appenders——输出源 ​

现在日志有了级别和类型，解决了日志入口处的级别 和 分类的问题，而在 log4js 中，即日志输出到哪里、以怎么样的方式输出，就是由appender来解决的，可以将日志写入到文件、发送电子邮件、通过网络发送数据等。

1. type：日志输出类型 ​

常用的类型：

  • console：输出到控制台
  • file：输出到指定的文件
  • dateFile：按时间分隔输出到不同的文件

2. 当 type: 'file' 时 ​

与它同级的属性：

属性
filename日志保存文件的路径及文件名，./为项目根目录，（'file.log'） 这样写默认为项目的根目录
maxLogSize?日志文件的最大大小(以字节为单位),如果未指定，则不会发生日志滚动
backups?日志滚动期间要保留的旧日志文件数，默认值为5 （只有type为 'file' 时，backups才生效 ）
compress?是否压缩backups
encoding?编码格式，默认为utf-8
layout?输出的样式
js
import { resolve, dirname } from "path";
import { fileURLToPath } from "url";

const __filename = fileURLToPath(import.meta.url);
const __dirname = dirname(__filename);

/**
 * @type {import("log4js").Configuration}
 */
const config = {
  appenders: {
    app: {
      type: "console"
    },
    fileout: {
      type: "file",
      filename: resolve(__dirname, "./logs/app.log"),
      maxLogSize: 200, // 单位字节
      backups: 5,
      encoding: "utf-8"
    }
  },
  categories: {
    app: {
      appenders: ["fileout", "app"],
      level: "DEBUG"
    }
  }
};

export default config;

3. 当 type: dateFile 时 ​

与其同级的属性有以下：

属性
filename
pattern?滚动日志的时间类型，默认为 .yyyy-MM-dd
encoding?编码格式，默认为utf-8
compress?是否压缩滚动日志文件
daysToKeep?如果此值大于零，则日志滚动期间将删除早于该天数的文件，默认为0
alwaysIncludePattern?在当前日志文件名中和滚动日志一样包括pattern
layout?输出样式

pattern是保存滚动日志的时间分割，它的规则如下：

  • yyyy : 年
  • MM : 月
  • dd : 日
  • hh : 时
  • mm : 分
  • ss : 秒
  • SSS : 毫秒
js
import { resolve, dirname } from "path";
import { fileURLToPath } from "url";

const __filename = fileURLToPath(import.meta.url);
const __dirname = dirname(__filename);

/**
 * @type {import("log4js").Configuration}
 */
const config = {
  appenders: {
    datefileout: {
      type: "dateFile",
      filename: resolve(__dirname, "./logs/app.log"),
      pattern: ".yyyy-MM-dd",
      alwaysIncludePattern: true,
      maxLogSize: 200,
      backups: 5,
      encoding: "utf-8"
    }
  },
  categories: {
    app: {
      appenders: ["datefileout"],
      level: "DEBUG"
    }
  }
};

export default config;

4. 格式化 ​

类型为 console 的 appenders 配置采用默认的格式—— basic。

appenders 包含 layout 字段，包含以下几个类型：

  • basic：包含时间戳、日志级别、日志类型基本日志格式。
  • colored：格式与basic一致，只是不同日志等级显示不同的颜色。
  • messagePassThrough：只输出传递的日志信息，没有时间戳、日志级别、日志类型。
  • dummy：只输出第一个参数的内容，没有时间戳、日志级别、日志类型。
  • pattern：可自定义日志格式。
  • custom-layout：自定义格式。

basic ​

js
/**
 * @type {import("log4js").Configuration}
 */
const config = {
  appenders: {
    basic: {
      type: "console",
      layout: {
        type: "basic"
      }
    },
  },
  categories: {
    default: {
      appenders: ["basic"],
      level: "DEBUG"
    }
  }
};

export default config;

pattern ​

传递模板字符串，字符串的规则为 %[padding?].[truncation?][field]{[format]}，其中1间距、2截断参数可选。

field 可以是以下任意字段：

  • %r，本地时间
  • %p，日志级别
  • %c，日志分类
  • %h，主机名称，比如 http://localhost:3000/ 的 localhost
  • %m，日志数据，也就是调用 logger.debug() 传递的所有数据
  • %m{l}，切割日志数据长度，相当于 arr.slice(startNum)
  • %m{l,u}，切割日志数据长度，相当于 arr.slice(startNum,endNum)
  • %d，时间格式，格式选项为 ISO8601 ： ISO8601 、 、 ISO8601_WITH_TZ_OFFSET ABSOLUTETIME 或 DATETIME 任何与日期格式库兼容的字符串。例如 %d{DATETIME} ， %d{yyyy/MM/dd-hh.mm.ss}
  • %%，当想要输出 % 符号时
  • %n，换行符
  • %z，进程 ID（来自 process.pid ）
  • %f，filename 的完整路径
  • %f{depth}， 路径的深度允许您选择仅具有文件名
  • %l，行号
  • %o，列位置
  • %s，调用堆栈
  • %C，类名称
  • %M，方法或函数名称
  • %A，方法或函数别名
  • %F，完全限定的调用方名称
  • %[，标志一个定制一个颜色区域的开始
  • %]，标志一个定制一个颜色区域的结束
js
pattern: {
  type: "console",
    layout: {
      type: "pattern",
        pattern: "%d %p %c %m%n"
    }
}

custom-layout ​

自定义样式布局，通过 log4js.addLayout 添加 layout ，然后使用

js
log4js.addLayout("json", (config) => {
  // config为appenders的layout配置
  return function (logEvent) {
    // logEvent 为log4j传递的事件对象，数据包含：
    // 开始时间、分类名称、log数据、日志级别、context上下文、pid进程id、error
    return JSON.stringify(logEvent) + config.separator;
  };
})

// config
customLayout: {
  type: "console",
    layout: {
      type: "json",
        separator: "," // 分隔符
    }
}
